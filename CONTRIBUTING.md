# Contributing

This is an internal Kokoro Tech repository. Contributions follow the engineering
governance framework defined in [kokoro-constitution](https://gitlab.com/kokoro-tech/governance/kokoro-constitution).

## Workflow

1. Create a branch from `main` using a conventional prefix: `feat/...`, `fix/...`, `docs/...`, `chore/...`, `refactor/...`.
2. Use [Conventional Commits](https://www.conventionalcommits.org/) for commit messages (`feat(scope): ...`, `fix(scope): ...`, etc.).
3. Open a merge request targeting `main`.
4. Ensure CI passes (once configured) and request review from a maintainer.

## Standards

Engineering standards are defined in [kokoro-constitution](https://gitlab.com/kokoro-tech/governance/kokoro-constitution):

- Coding standards
- Testing standards (**KT-TEST-001**)
- Documentation standards (**KT-DOCS-001**)
- Development execution modes (**KT-DEVMODE-001**)
- Technical design proposals (**KT-RFC-001**)
- Operational documentation (**KT-OPS-002**)

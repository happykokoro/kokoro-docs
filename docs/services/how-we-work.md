# How We Work

**Practical answers to common questions about engaging Kokoro Tech.**

---

## Engagement Process

Every project follows a six-phase structure designed to minimize surprises and maximize the quality of the delivered system.

| Phase              | Description                                                                                                                   |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------- |
| **1. Discovery**   | We review your requirements, ask clarifying questions, and assess technical fit. Usually 1–3 business days.                   |
| **2. Proposal**    | We deliver a written proposal: scope, deliverables, timeline, milestones, and pricing. No vague estimates.                    |
| **3. Development** | Active build phase. You receive weekly progress reports and can review work at each milestone.                                |
| **4. Review**      | You test the delivered system. We address feedback and fix issues within the agreed scope.                                    |
| **5. Delivery**    | Final handoff: source code, deployment instructions, documentation, and a working production system.                          |
| **6. Support**     | Post-delivery support period (duration specified in proposal). Bug fixes at no charge; scope additions are scoped separately. |

## IP Ownership

**Clients own all deliverables.** Full source code, all rights, no license fees. The code we write for your project belongs to you.

We retain the right to describe the engagement category in our portfolio (e.g., "built a DeFi liquidation bot") without disclosing client identity or proprietary details, unless you prefer otherwise.

## Communication

- **Primary channel**: GitHub Issues or a shared project board — async-first, fully documented
- **Secondary**: Discord or Telegram for rapid questions
- **Weekly sync calls**: 30 minutes, optional but recommended for multi-month engagements
- **Response time**: Within 48 business hours for all inquiries

We are based in Asia-Pacific (UTC+8) but operate globally. Async communication means timezone mismatches are rarely a bottleneck.

---

## Technology Selection

We recommend technology based on requirements, not preference. If your team needs to maintain the system after delivery, we factor in your team's existing skills. If you need maximum performance, we recommend Rust. If you need rapid iteration, TypeScript or Python may be more appropriate.

We will tell you honestly when a technology choice is wrong for your use case — including our own preferred stack.

## AI-Augmented Delivery

Kokoro Tech uses AI-augmented development as a primary engineering methodology. Here is what that means for clients:

**What it means for you:**

- Faster delivery — parallel research, implementation, and review phases
- Lower per-feature cost — AI parallelism eliminates serial development bottlenecks
- Higher consistency — AI-enforced coding conventions across the entire codebase
- Same quality bar — zero-warnings, clippy-clean, full test suite required regardless of who wrote the code

**What it does not mean:**

- We do not ship AI-generated code without review and testing
- The same CI/CD gates (build, lint, format, test) apply to all code
- Client-specific domain knowledge is always human-contributed

The 115 MCP tools and Agent Orchestra platform used to build the Kokoro platform are applied to client engagements when scope and timeline warrant it.

---

## What We Don't Do

- We don't build systems we cannot support — if your requirements exceed our domain knowledge, we'll say so upfront
- We don't take projects where we'd be learning a technology from scratch at your expense
- We don't work without a written scope — "figure it out as we go" engagements consistently disappoint everyone
